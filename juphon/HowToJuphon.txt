To run a phonon calculation at the moment, do the following:

1) Write a suitable input file for the input generator. Set the k-point generation
   to a grid of N x N x N.
2) Run inpgen as follows: /path/to/fleur/inpgen -f inputFile
2.1) Run inpgen again with: /path/to/fleur/inpgen -f inputFile -nosym -add_name desym
     This will generate an inp.xml with no symmetries and exactly N^3 k-points that is distinguishable from the normal one.
3) Run inpgen again with /path/to/fleur/inpgen -inp.xml -noKsym -kpt dfpt#grid=N,N,N
   to generate an additional k-Point-Grid that covers the full Brillouin zone.
3.1) Run inpgen one more time with /path/to/fleur/inpgen -inp.xml -noKsym -add_name desym -kpt dfpt#grid=N,N,N
     for the desymmetrized input.
4) Set the xc-functional tag to an LDA from libxc (e.g. VWN):
      <xcFunctional name="LibXC" relativisticCorrections="F">
         <LibXCName  exchange="lda_x" correlation="lda_c_vwn"/>
      </xcFunctional>
5) Run Fleur normally [/path/to/fleur/fleur(_MPI)] and converge out.
[5.1) Optimize the ground state energy w.r.t. the lattice parameter.]
[5.2) Ideally: Back up the result.]
6) Modify the inp.xml: Set <<<numbands="all">>>, <kPointListSelection listName="dfpt"/>
   and add the tag <juPhon l_dfpt="T" singleQpt="M"/> in the output section [M corresponds
   to the index of the desired q-Point in kpts.xml @dfpt] or, beforehand, set a proper
   q-point path in the source code [input is WIP].
7) Run Fleur. The result should be the dynamical matrix, its Eigenvectors and Eigenvalues
   [additionally transformed into frequencies].
X) For the next q-Point[if singleQpt used]: Delete the folder's contents, retrieve the backup and repeat
   steps 6+7 for the next q-Point index.
